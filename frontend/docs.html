Documentação da API Paneas

Bem-vindo à documentação oficial da API Paneas Ecosystem. Esta API oferece componentes completos de IA para transcrição, análise, conversação e síntese de voz.

Stack 100% Local

Todos os modelos rodam localmente, garantindo máxima privacidade e controle dos seus dados.

Autenticação

Atualmente a API não requer autenticação para uso local. Para ambientes de produção, recomendamos implementar autenticação via Bearer Token.

Exemplo com Bearer Token (futuro) 
curl -X POST http://localhost:8765/api/v1/chat/completions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{...}'

Base URL

Todas as requisições devem ser feitas para a seguinte URL base:

http://localhost:8765/api/v1

POST /chat/completions

API compatível com OpenAI para conversação com o modelo Paneas-32B (Qwen 2.5). Suporta modo streaming para respostas em tempo real.

Parâmetros

Nome Tipo Obrigatório Descrição
model string Sim Nome do modelo: "paneas-q32b"
messages array Sim Array de mensagens no formato OpenAI
temperature float Não Controla aleatoriedade (0.0-2.0). Padrão: 0.7
max_tokens integer Não Máximo de tokens na resposta. Padrão: 2048
stream boolean Não Ativar streaming. Padrão: false

Exemplo de Requisição

cURL 
curl -X POST http://localhost:8765/api/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "paneas-q32b",
    "messages": [
      {
        "role": "system",
        "content": "Você é um assistente prestativo."
      },
      {
        "role": "user",
        "content": "Explique o que é inteligência artificial"
      }
    ],
    "temperature": 0.7,
    "max_tokens": 500
  }'

Exemplo de Resposta

JSON Response 
{
  "id": "chatcmpl-123",
  "object": "chat.completion",
  "created": 1677652288,
  "model": "paneas-q32b",
  "choices": [
    {
      "index": 0,
      "message": {
        "role": "assistant",
        "content": "Inteligência Artificial (IA) é um ramo da ciência da computação..."
      },
      "finish_reason": "stop"
    }
  ],
  "usage": {
    "prompt_tokens": 20,
    "completion_tokens": 150,
    "total_tokens": 170
  }
}

POST /asr

Transcrição de áudio usando Whisper. Suporta múltiplos formatos e detecção automática de idioma.

Parâmetros

Nome Tipo Obrigatório Descrição
file file Sim Arquivo de áudio (mp3, wav, m4a, ogg)
language string Não Código do idioma (pt, en, es). Auto-detect se omitido

Exemplo de Requisição

cURL 
curl -X POST http://localhost:8765/api/v1/asr \
  -F "file=@audio.mp3" \
  -F "language=pt"

Exemplo de Resposta

JSON Response 
{
  "text": "Olá, bem-vindo ao sistema de transcrição automática.",
  "language": "pt",
  "duration": 5.2,
  "segments": [
    {
      "start": 0.0,
      "end": 5.2,
      "text": "Olá, bem-vindo ao sistema de transcrição automática."
    }
  ]
}

POST /diar

Separação de speakers (diarização) com suporte a LLM. O modo LLM é mais rápido e otimizado para conversas entre 2 speakers (Atendente/Cliente).

Modo LLM Recomendado

Use use_llm=true para diarização rápida e precisa em conversas de call center.

Parâmetros

Nome Tipo Obrigatório Descrição
file file Sim Arquivo de áudio
use_llm boolean Não Usar LLM para diarização. Padrão: true
num_speakers integer Não Número de speakers (modo tradicional apenas)

Exemplo de Requisição - Modo LLM

cURL 
curl -X POST http://localhost:8765/api/v1/diar \
  -F "file=@conversation.mp3" \
  -F "use_llm=true"

Exemplo de Resposta

JSON Response 
{
  "mode": "llm",
  "conversation": [
    {
      "speaker": "Atendente",
      "text": "Bom dia, meu nome é Carlos, como posso ajudar?"
    },
    {
      "speaker": "Cliente",
      "text": "Olá, gostaria de informações sobre planos."
    }
  ]
}

POST /tts

Síntese de voz (Text-to-Speech) com vozes clonadas e controle de velocidade.

Parâmetros

Nome Tipo Obrigatório Descrição
text string Sim Texto para sintetizar
voice string Não Nome da voz. Padrão: voz padrão
speed float Não Velocidade (0.5-2.0). Padrão: 1.0

Exemplo de Requisição

cURL 
curl -X POST http://localhost:8765/api/v1/tts \
  -H "Content-Type: application/json" \
  -d '{
    "text": "Olá, bem-vindo ao Paneas Ecosystem",
    "voice": "pt-BR",
    "speed": 1.0
  }' \
  --output audio.wav

Streaming (Server-Sent Events)

O endpoint /chat/completions suporta streaming de respostas usando Server-Sent Events (SSE). Isso permite que você receba a resposta do modelo token por token em tempo real.

JavaScript / Fetch API

JavaScript 
const response = await fetch('http://localhost:8765/api/v1/chat/completions', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    model: 'paneas-q32b',
    messages: [{role: 'user', content: 'Conte uma história'}],
    stream: true
  })
});

const reader = response.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const {done, value} = await reader.read();
  if (done) break;

  const chunk = decoder.decode(value);
  const lines = chunk.split('\n');

  for (const line of lines) {
    if (line.startsWith('data: ')) {
      const data = line.slice(6);
      if (data === '[DONE]') break;

      const parsed = JSON.parse(data);
      const content = parsed.choices[0].delta.content || '';

      // Processa cada token recebido
      process.stdout.write(content);
    }
  }
}

Python

Python 
import requests
import json

response = requests.post(
    'http://localhost:8765/api/v1/chat/completions',
    json={
        'model': 'paneas-q32b',
        'messages': [{'role': 'user', 'content': 'Conte uma história'}],
        'stream': True
    },
    stream=True
)

for line in response.iter_lines():
    if line.startswith(b'data: '):
        data = line[6:].decode('utf-8')
        if data == '[DONE]':
            break

        parsed = json.loads(data)
        content = parsed['choices'][0]['delta'].get('content', '')
        print(content, end='', flush=True)

Tratamento de Erros

A API retorna erros HTTP padrão com detalhes em JSON.

Códigos de Status HTTP

Código Significado Descrição
200 OK Requisição bem-sucedida
400 Bad Request Parâmetros inválidos ou ausentes
422 Unprocessable Entity Dados válidos mas não processáveis
500 Internal Server Error Erro interno do servidor
503 Service Unavailable Serviço temporariamente indisponível

Formato de Erro

JSON Error Response 
{
  "detail": "Descrição do erro",
  "error_code": "INVALID_PARAMETER",
  "field": "messages"
}
